# config/

Загрузка и валидация переменных окружения при старте приложения.

## Структура

```
config/
├── env.schema.ts   # Zod-схема всех env-переменных
└── env.ts          # Парсинг process.env → типизированный объект `env`
```

## Назначение

Приложение **не читает `process.env` напрямую** нигде, кроме `env.ts`. Весь остальной код импортирует готовый объект:

```typescript
import { env } from '@/config/env';
```

При невалидных переменных процесс падает при старте с понятным сообщением об ошибке.

## Кодстайл

- Каждая новая env-переменная добавляется **одновременно** в `env.schema.ts` и `.env.example`
- Булевы env парсятся через кастомный `envBoolean` preprocessor (`'1'`/`'true'`/`'yes'`/`'on'` → `true`)
- Числовые env используют `z.coerce.number()` для автоматической конвертации строки → число
- Дефолты задаются через `.default()` в Zod-схеме, а не в коде
- Тип `Env` экспортируется из `env.schema.ts` как `z.infer<typeof envSchema>`
